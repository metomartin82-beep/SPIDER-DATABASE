Why isolated databases

One real bug shouldn't slow down the other nine thousand lines of perfect code.

A shared database with a tenant_id column works fine until it doesn't. SpiderDB skips the risk entirely — every user gets a real, separate database from the start.

True isolation

Every database is a separate instance with its own storage. No shared tables, no row-level security to get wrong, no cross-tenant query leaking into logs.

Instant provisioning

Create a database in the time it takes to make an API call — no queues, no waiting on infrastructure. Spin one up per signup, per session, or per AI agent run.

SQLite-compatible

Connect with any SQLite or libSQL client you already know. No new query language, no ORM plugin, no migration off tools your team already trusts.

Automatic backups

Every database is backed up on a rolling schedule with point-in-time restore, without a single line of backup code in your app.

Branch and clone

Fork any database to test a migration or debug production data, without touching the original. Delete the branch when you're done.

Scoped access tokens

Generate a read-only or read-write token bound to one database, with an expiry date, and hand it to a client or a background job without exposing the rest of your fleet.

How it works

Three calls between you and a running database.

Create

Call the API or run one CLI command. SpiderDB provisions an isolated database and returns a connection URL.

Connect

Use the connection string with any SQLite/libSQL client — no driver to install that you don't already have.

Ship

Query it like any local SQLite file. Backups, replication, and isolation are handled for you in the background.

Try it now

Create a database from your terminal.

# install the CLI npm install -g spiderdb # authenticate once spiderdb login # spin up a database for a new user spiderdb databases create user_9f21 ✓ created user_9f21 libsql://user-9f21.spiderdb.io
import { SpiderDB } from "spiderdb"; const client = new SpiderDB({ token: process.env.SPIDERDB_TOKEN }); const db = await client.databases.create("user_9f21"); console.log(db.connectionUrl); // libsql://user-9f21.spiderdb.io
from spiderdb import SpiderDB client = SpiderDB(token=os.environ["SPIDERDB_TOKEN"]) db = client.databases.create("user_9f21") print(db.connection_url) # libsql://user-9f21.spiderdb.io
