Platform API

The Platform API is a REST API for managing databases programmatically. All requests require an Authorization: Bearer header with an account-level API key, generated from the dashboard under Settings → API Keys.

Base URL: https://api.spiderdb.io/v1

List databases

GET /v1/databases
curl https://api.spiderdb.io/v1/databases \
  -H "Authorization: Bearer $SPIDERDB_API_KEY"
{
  "databases": [
    {
      "name": "customer_9f21",
      "region": "iad",
      "status": "active",
      "createdAt": "2026-06-02T10:14:00Z"
    }
  ]
}

Create a database

POST /v1/databases
FieldTypeRequiredDescription
namestringyesUnique name, lowercase letters/numbers/hyphens/underscores only.
groupstringnoGroup to create the database in. Defaults to "default".
regionstringnoPrimary region. Defaults to your account's home region.
 
curl -X POST https://api.spiderdb.io/v1/databases \
  -H "Authorization: Bearer $SPIDERDB_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "customer_9f21"}'
{
  "name": "customer_9f21",
  "connectionUrl": "libsql://customer-9f21.spiderdb.io",
  "region": "iad",
  "status": "active",
  "createdAt": "2026-07-14T09:02:11Z"
}

Get a database

GET /v1/databases/:name
curl https://api.spiderdb.io/v1/databases/customer_9f21 \
  -H "Authorization: Bearer $SPIDERDB_API_KEY"

Delete a database

DELETE /v1/databases/:name
This is permanent. Deleting a database also deletes its backups after the retention window on your plan. Export any data you need first.
curl -X DELETE https://api.spiderdb.io/v1/databases/customer_9f21 \
  -H "Authorization: Bearer $SPIDERDB_API_KEY"

Pause / resume a database

POST /v1/databases/:name/pause
POST /v1/databases/:name/resume

A paused database keeps its storage and stops counting toward your active read/write quota. Resuming is near-instant.

Clone a database

POST /v1/databases/:name/clone
curl -X POST https://api.spiderdb.io/v1/databases/customer_9f21/clone \
  -H "Authorization: Bearer $SPIDERDB_API_KEY" \
  -d '{"newName": "customer_9f21_staging"}'

Clones are copy-on-write and share underlying storage with the source database until either one is modified, so cloning is fast regardless of database size.

Database tokens

POST /v1/databases/:name/tokens
FieldTypeDescription
permissionstring"read-only" or "read-write". Defaults to "read-write".
expiresInstringDuration like "24h" or "30d". Omit for a non-expiring token.
 
curl -X POST https://api.spiderdb.io/v1/databases/customer_9f21/tokens \
  -H "Authorization: Bearer $SPIDERDB_API_KEY" \
  -d '{"permission": "read-only", "expiresIn": "30d"}'
{
  "token": "eyJhbGciOiJFZERTQSJ9...",
  "permission": "read-only",
  "expiresAt": "2026-08-13T09:02:11Z"
}

Errors

Errors are returned as JSON with an HTTP status code and a machine-readable code field.

StatusCodeMeaning
400invalid_nameDatabase name doesn't match the allowed pattern.
401unauthorizedMissing or invalid API key.
404not_foundNo database with that name exists on this account.
409already_existsA database with that name already exists.
429plan_limit_reachedYou've hit your plan's database or usage limit.
