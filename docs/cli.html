CLI Reference

The SpiderDB CLI wraps the Platform API for use from your terminal or CI pipelines.

Install

npm install -g spiderdb

Requires Node.js 18 or later. Verify the install:

spiderdb --version

spiderdb login

Opens your browser to authenticate and stores a credential in ~/.spiderdb/credentials.

spiderdb login

For CI environments without a browser, use an API key instead:

spiderdb login --token $SPIDERDB_API_KEY

spiderdb databases

CommandDescription
spiderdb databases listList all databases in your account.
spiderdb databases create <name>Create a new database.
spiderdb databases show <name>Show details for one database.
spiderdb databases destroy <name>Permanently delete a database.
spiderdb databases pause <name>Pause a database.
spiderdb databases resume <name>Resume a paused database.
spiderdb databases clone <name> <new-name>Clone a database.
spiderdb databases rename <name> <new-name>Rename a database.
 
$ spiderdb databases list

NAME              REGION   STATUS   CREATED
customer_9f21     iad      active   2 days ago
customer_a02e     iad      active   2 days ago
customer_beta     fra      paused   1 week ago

spiderdb shell

Opens an interactive SQL shell against a database — useful for quick inspection without writing a script.

spiderdb shell customer_9f21

spiderdb> SELECT COUNT(*) FROM users;
spiderdb> .tables
spiderdb> .schema users
spiderdb> .quit

spiderdb tokens

CommandDescription
spiderdb tokens create <db> [--readonly] [--expiry 30d]Create a scoped token for a database.
spiderdb tokens list <db>List active tokens for a database.
spiderdb tokens revoke <db> <token-id>Revoke a token immediately.
Tip: revoke every token for a database if you suspect one has leaked — this is faster than rotating them individually and takes effect immediately.
